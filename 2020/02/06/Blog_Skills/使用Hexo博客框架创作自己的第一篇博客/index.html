使用Hexo博客框架创作自己的第一篇博客

发表于 2020-02-06

使用Markdown，Hexo和Github进行博客创作。

创作自己的第一篇博客

首先，介绍一下需要用到的3个工具：Markdown，Hexo，Github。

Markdown 是一款text-to-HTML的转换工具，其允许作者使用一种易于阅读、易于撰写的纯文本格式进行写作，然后将该文本转化为有效的XHTML（或HTML）格式。因此，可以将” Markdown “看成两个东西：（1）一种纯文本语法格式。（2）一个软件工具，可将纯文本格式转换为HTML格式。

Hexo 是一个博客框架，其可以将基于 Markdown 撰写的文章生成美观的静态网页。

Github 是一个代码托管平台，用于软件项目的版本控制和协作。我们可以直接从 Github 仓库使用 Github Pages 托管自己的网站。Github Pages 是一项静态站点托管服务，它可以直接从 Github 仓库获取HTML, CSS和JavaScript文件，并发布到网站。

基于这3个工具撰写并发表个人博客的工作流程大致如下：先用 Hexo 工具新建一个博客，同时会自动创建一个 Markdown 文件，然后用MarkdownPad软件编辑该 Markdown 文件进行博客撰写，最后用 Hexo 工具将撰写好的博客生成静态网页并部署到 Github 。具体地：

  • 第一步：新建博客，$ hexo new <title>。
  • 第二步：编辑博客，利用 MarkdownPad 软件编辑source/_posts/文件下在上一步中自动创建的.md文件。
  • 第三步：生成本地静态网页，$ hexo generate。
  • 第四步：将本地静态网页部署到 Github，$ hexo deploy。

最后，完成以上四步就可以在username.github.io网页上查看博客了，其中username是你的Github用户名。

注意事项：

  • 在首次新建博客之前，运行$ hexo init [folder]初始化一个网站，即生成一个hexo项目文件夹。只有在该项目文件夹下才能运行运行$ hexo new <title>。
  • 生成本地静态网页之后，运行$ hexo serve可在localhost:4000上预览静态网页，默认端口为4000。 运行$ hexo serve -p 5000可修改静态网页端口为5000。运行 $ hexo help serve可查看更多用法。
  • 将静态网页部署到Github上之后，如果网站username.github.io上没有全部更新，运行$ hexo clean清除缓存后再运行$ hexo deploy进行网站部署。

准备工作

这里补充介绍结合Hexo和Github搭建个人博客的准备工作。从零开始搭建的同学可以从这里可是动手实践。

注册Github账号

在Github官网Sign up一个账号即可。注意，注册的邮箱要验证，后面会用到这个邮箱。

创建仓库

新建一个名为username.github.io的仓库(注意，只能使用这种命名方式)，其中username为你的用户名，填写其他名称无效。将来你的个人博客地址就是https://username.github.io/。

安装和配置Git

根据Git官网的步骤安装即可。Window用户可以打开https://git-scm.com/download/win下载合适的Git版本，如Git-2.19.1-64-bit.exe。点击下载的安装包，一直next，直到安装完成。此时右键点击桌面，你会看到Git Bash Here选项，点击该选项，则会弹出终端界面。在终端中运行$ git --version可查看Git的版本信息。

至此，Git安装和配置完毕。

安装Node.js

参考Hexo官网官网：

Node.js 为大多数平台提供了官方的 安装程序。对于中国大陆地区用户，可以前往 淘宝 Node.js 镜像 下载。

其它的安装方法：

  • Windows：通过 nvs（推荐）或者nvm 安装。
  • Mac：使用 Homebrew 或 MacPorts 安装。
  • Linux（DEB/RPM-based）：从 NodeSource 安装。
  • 其它：使用相应的软件包管理器进行安装，可以参考由 Node.js 提供的 指导

对于 Mac 和 Linux 同样建议使用 nvs 或者 nvm，以避免可能会出现的权限问题。

Windows 用户
使用 Node.js 官方安装程序时，请确保勾选 Add to PATH 选项（默认已勾选）

For Mac / Linux 用户

如果在尝试安装 Hexo 的过程中出现 EACCES 权限错误，请遵循 由 npmjs 发布的指导 修复该问题。强烈建议 不要 使用 root、sudo 等方法覆盖权限

安装Hexo

按照Hexo官网步骤操作即可。

$ npm install hexo-cli -g
$ hexo init blog // 生成hexo项目文件夹
$ cd blog
$ npm install
$ hexo server

至此，就可以按照创作自己的第一篇博客小节中的四个步骤开始博客创作之旅了。
